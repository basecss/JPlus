Buttons

Button Class Description
Default .x-button Standard gray button with gradient
Primary .x-button-primary Provides extra visual weight and identifies the primary action in a set of buttons
Info .x-button-info Used as an alternate to the default styles
Success .x-button-success Indicates a successful or positive action
Warning .x-button-warning Indicates caution should be taken with this action
Danger .x-button-danger Indicates a dangerous or potentially negative action

Buttons for actions

As a convention, buttons should only be used for actions while hyperlinks are to be used for objects. For instance, "Download" should be a button while "recent activity" should be a link.

For anchors and forms

Button styles can be applied to anything with the .x-button applied. However, typically you'll want to apply these to only <a> and <button> elements.

Note: All buttons must include the .x-button class. Button styles should be applied to <button> and <a> elements for consistency.

Cross browser compatibility

In IE9, we drop the gradient on all buttons in favor of rounded corners as IE9 doesn't crop background gradients to the corners.

Related, IE9 jankifies disabled button elements, rendering text gray with a nasty text-shadow—unfortunately we can't fix this.
