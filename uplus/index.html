DPL 项目规范

在制作和使用DPL组件时，必须先读此文。

一、名词

  • DPL(Design Pattern Libaray): 设计模式库。
  • 组件(Component)：DPL的组成部分，DPL 内的每个元素都称为组件，一个组件是一套功能的集合。一个组件内可以包含子组件。两个组件之间也可以互相依赖。
  • 控件(Control)：控件是专门负责界面功能的组件。控件本身是和业务无关的。
  • 配件(Widget)：专门为业务准备的组件 。
  • 框架(Framework)：专门为解决一类目标而写的代码统称为框架。
  • 私有文件（夹）：开发DPL时所使用的、不发布的文件（夹）叫私有文件（夹），私有文件（夹）在命名上以下划线（_）开头。
  • 临时文件（夹）：临时使用的或由程序生成的文件（夹），私有文件（夹）在命名上以波浪线（~）开头。

三、文件夹格式

每个组件的结构以 _tpl 文件夹的结构为基准。

  • _tpl: 一个组件所必须的文件夹。
    • assets: 组件所需用的静态资源文件。
      • scripts: 组件所需用的 Javascript 文件或其它脚本文件（如 .htc 文件）
      • images：组件所需用的图片、图标和多媒体文件。
      • styles: 组件所需用的 CSS 样式文件或其它样式文件（如 .theme 文件)
    • index.html: 组件的示例页。
如果一个组件包含多个子组件，可以在组件跟目录新建一个子组件名的 html 文件。并在 assets 相应位置新建同名文件。 如：添加一个 button 子组件，应该新建以下 3 个文件： button.html assets/scripts/button.js assets/styles/button.css 文件和文件夹名字必须为全小写。

四、CSS 命名规范

为防止组件之间的冲突，必须对组件类命名作限制：

CSS 命名原则：

  1. 组件跟节点的 CSS 命名必须是 b-当前组件名。
  2. 组件内部节点的 CSS 命名必须以 b-当前组件名- 开头。
  3. 使用多个类名，而不是一个类多个功能。(即使用 .a-b 而不是使用 .a.b)

C如 button 组件：

HTML 代码：

<a href="#"><span>文字</span></a>

正确的 CSS 命名：

<a href="#" class="b-button"><span class="b-button-content">文字</span></a>

错误的 CSS 命名：

<a href="#" class="b-button"><span class="b-content">文字</span></a> <a href="#" class="b-button-outer"><span class="b-button-content">文字</span></a> <a href="#" class="b-btn"><span class="b-btn-cnt">文字</span></a>

如 grid 组件：

HTML 代码：

<div>文字</div>

正确的 CSS 命名：

<div class="b-grid b-grid-2"></div>

错误的 CSS 命名：

<div class="b-grid-2"></div> <div class="b-grid b-2"></div>

五、 Javascript 组件规范

  1. 对象名必须和组件名一样，且首字母大写。
  2. 内部传递原生 DOM 对象，而非通过 ID、CSS 选择器 或 其它方式。
